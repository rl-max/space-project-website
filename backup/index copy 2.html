SPACE enables learning and deployment across heterogeneous robots and can adapt to new dynamics at test time, such as increased control Hz and heavy object weights.

Motivation

🔒 Discrepancies in Actions across different robot setups

Replaying actions collected from one robot to the other robot results in different trajectory.

🤔 Why?: Actions are defined as underlying robot control command

Different controller implementations across embodiments affect the robot motion for the same action. Actions are also affected by hardware wear and tear, and manufacturing variability in the same embodiment.

❗ This significantly hurts learning & deployment across different robots.

Our framework: SPACE

🔑 Predict Cartesian state delta, an actual robot movement during data collection.
